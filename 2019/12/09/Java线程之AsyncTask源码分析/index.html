Java线程之AsyncTask源码分析

发表于 2019-12-09

简介

  • 一个Android 已封装好的轻量级异步类
  • 属于抽象类，即使用时需 实现子类
  • 实现多线程，在工作线程中执行任务，如耗时任务
  • 实现工作线程 & 主线程（UI线程）之间的通信
  • 方便实现异步通信
  • 采用线程池的缓存线程 + 复用线程，避免了频繁创建 & 销毁线程所带来的系统资源开销

分别在主线程以及子线程里调用：

构造方法

构造方法中初始化了三个变量，mHandler，mWorker，mFuture；

  • mHanler：取自任务执行线程的Looper，用于在主线程和子线程中调度任务
  • mWoker：在调用线程中封装参数，准备通过调用mHandler传递给子线程
  • mFuture：封装任务执行结束的结果，准备通过sHandler(获取的是主线程的Looper)传递给主线程

线程池：AsyncTask封装了两种线程池用于不同的场景

  • 调用task.execute方法使用sDefaultExecutor来执行线程，sDefaultExecutor的实现，该方法调用的任务同步执行
  • 调用task.executeOnExecutor(AsyncTask.THREAD_POOL_EXECUTOR)，使用一个核心线程，最多20个并发线程执行任务
  • 如果并发数过多，会启用拒绝策略sRunOnSerialPolicy,启用备用线程池sBackupExecutor？（测试未发现被调用，可能方式不对）
