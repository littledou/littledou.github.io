Android源码分析<3>网络请求库Volley

发表于 2019-12-17

摘自 官网 描述

Volley 是一个可让 Android 应用更轻松、（最重要的是）更快捷地联网的 HTTP 库。您可以在 GitHub 上获取 Volley。

Volley 具有以下优势：

  • 自动网络请求调度。
  • 多个并发网络连接。
  • 透明磁盘和具有标准 HTTP 缓存一致性的内存响应缓存。
  • 支持请求优先级。
  • 取消请求 API。您可以取消单个请求，也可以设置要取消的请求的时间段或范围。
  • 可轻松自定义，例如自定义重试和退避时间。
  • 强大的排序功能，让您可以轻松使用从网络异步提取的数据正确填充界面。
  • 调试和跟踪工具。

Volley 在用于填充界面的远程过程调用 (RPC) 类型的操作方面表现出色，例如以结构化数据的形式获取搜索结果页面。它可以轻松集成任何协议，并且开箱后即支持原始字符串、图片和 JSON。Volley 提供对您所需功能的内置支持，因此您无需编写样板代码，且可以专注于特定于应用的逻辑。

Volley 不适用于下载大量内容的操作或流式传输操作，因为在解析过程中，Volley 会将所有响应存储在内存中。对于下载大量内容的操作，请考虑使用 DownloadManager 等替代方法。

应用

因为OkHttp的势力庞大以及应用广泛，Volley基本上不再作更新。但其功能完善，代码逻辑简单，供学习还是及其丰富的。

生命周期分析

先来张请求的生命周期示例图，同样来自 官网。可以看到Volley将网络请求流程分为3个线程：

  • 主线程
  • 缓存线程
  • 网路请求执行线程

主线程发起请求，将请求（Request）调度至缓存线程/网络线程。
缓存线程处理请求，如无当前缓存或缓存过期，则将请求调度至网络线程，如命中缓存，则将响应结果调度至主线程。
网路线程接收到Request，发起Http请求，得到响应后将响应调度至主线程。
整个流程极为简洁，没有看到任何炫技以及多余，并且功能非常强大。这才是一个优化完善的SDK。

创建请求

Volley原生支持StringRquest、JsonArrayRequest、JsonObjectRequest以及ImageRequest。也可以通过自行实现parseNetworkResponse方法来自定义Request，格式化出自己需求的数据格式。

Request实现了Comparable接口，compareTo方法表明优先级高的Requset先执行，优先级相同的根据队列的先入先出规则执行。

构建请求队列

通过Volley类管理请求队列的创建。API9以上统一使用HttpURLConnection发起请求

RequestQueue

  • 初始化网络请求队列、缓存队列PriorityBlockingQueue
  • 有线程池的请求队列，用于调度任务，默认启用4个线程用于执行网络请求。额外启用一个缓存处理线程。
  • 线程启动后，将任务添加至响应的队列处理
任务调度

启动线程后，将任务添加至队列。

缓存队列处理

  • 计算缓存key，当缓存不存在，将任务丢还给mNetworkQueue
  • 若缓存命中，校验是否已过期，通过ExecutorDelivery将响应返回至主线程
