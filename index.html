JAVA数据结构NIO之Buffer

发表于 2019-12-10


NIO中 Buffer 是一个用于存储基本类型值的容器，它以类似于数组的方式存储和组织数据，线性有序，每个基本数据类型都有一个子类与之对应。

##

参数、方法

0 <= mark <= position <= limit <= capacity

  • capacity:容量
  • limit: 限制, 限制索引，数据只能读取到当前索引前一位，不包括当前索引
  • position: 位置
  • mark: 标记
  • remaining(): 返回当前位置与limit之间的元素个数（limit - position）
  • mark(): 标记当前position，在调用reset后将position恢复到前一个mark点
  • clear(): 还原缓冲区的状态，一般用于清空后重新存数据
  • flip(): 对缓冲区反转，缩小limit的范围至position，position归0，丢弃mark，用于截取部分
  • hasArray(): 是否存在底层实现的数组，直接缓冲区不存在
  • rewind(): 重绕缓冲区，position归0，丢弃mark，一般在重新读取缓冲区数据时使用
  • arrayOffset(): 偏移量

Java线程之AsyncTask源码分析

发表于 2019-12-09

简介

  • 一个Android 已封装好的轻量级异步类
  • 属于抽象类，即使用时需 实现子类
  • 实现多线程，在工作线程中执行任务，如耗时任务
  • 实现工作线程 & 主线程（UI线程）之间的通信
  • 方便实现异步通信
  • 采用线程池的缓存线程 + 复用线程，避免了频繁创建 & 销毁线程所带来的系统资源开销

分别在主线程以及子线程里调用：

构造方法

构造方法中初始化了三个变量，mHandler，mWorker，mFuture；

  • mHanler：取自任务执行线程的Looper，用于在主线程和子线程中调度任务
  • mWoker：在调用线程中封装参数，准备通过调用mHandler传递给子线程
  • mFuture：封装任务执行结束的结果，准备通过sHandler(获取的是主线程的Looper)传递给主线程
